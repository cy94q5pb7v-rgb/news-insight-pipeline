# news-insight-pipeline

Референс-код двух связанных конвейеров реальной системы новостной аналитики для
**банковско-travel домена** (премиальный банкинг + путешествия + ИИ-сервисы в
финансах). Код извлечён из большей рабочей системы, очищен и опубликован «как
есть» — чтобы сторонний разработчик мог понять архитектуру и переиспользовать
отдельные части.

Что здесь есть:

1. **Новостной конвейер** — «поиск ссылок → разрешение редиректов → классификация
   тем → извлечение текста статей → суммаризация и присвоение рейтингов через
   LLM-агента → дедупликация → загрузка в БД портала».
2. **Генерация инсайтов** — «новости + загруженные материалы → гипотезы/инсайты
   через LLM-агента → жизненный цикл гипотез → отображение на портале».

> **Термины коротко.** *Конвейер (pipeline)* — цепочка шагов, каждый берёт результат
> предыдущего, как конвейер на заводе. *LLM* (Large Language Model) — большая
> языковая модель (например, тип ChatGPT). *LLM-агент* — обёртка, которая
> запускает такую модель с инструментами и возвращает ответ. *Скоринг* — присвоение
> числовой оценки. *Дедупликация (dedupe)* — схлопывание дубликатов.

---

## Что НЕ входит

- Веб-приложение целиком (FastAPI-сервер, аутентификация, фронтенд-шаблоны
  кроме одного UI-файла инсайтов), бинарь `ru-fetch` (anti-bot прокси-клиент),
  Docker-обвязка SearXNG, cron-конфигурация, реальные базы данных и архивы.
- Сама LLM-модель. Все вызовы идут через CLI-обёртку (`core/llm_agent.py` +
  внешний бинарь `openclaw`). Её надо заменить на свой вызов модели — см.
  раздел Reuse и `docs/reuse-guide.md`.
- Внутренние модули, на которые ссылаются импорты `from web_app import ...`,
  `from core.auth import ...`, `from core.users import ...`, `from templates
  import ...`, `import orchestrator`. Их в этом репозитории нет — это швы для
  адаптации.

## Ключевые зависимости

**Стандартная библиотека Python** (3.10+; код использует `str | None`,
`dict[str, float]` — это синтаксис 3.10+): `urllib`, `xml.etree.ElementTree`,
`concurrent.futures`, `sqlite3`, `json`, `re`, `hashlib`, `subprocess`,
`threading`, `asyncio`, `heapq`, `base64`, `email.utils`.

**Внешние Python-библиотеки** (выведено из импортов):

| Библиотека | Где | Зачем |
|---|---|---|
| `trafilatura` | article_fetch, r1_fetch_urls, r1_prefetch | основной извлекатель текста статей |
| `beautifulsoup4` + `lxml` | r1_fetch_urls | парсинг HTML-листингов, очистка от навигации |
| `googlenewsdecoder` | r1_fetch_urls | декодирование новых HMAC-URL Google News без браузера |
| `fastapi` | kb_db, kb_ingest, routes_* | веб-API (внешний для этого репо слой) |
| `pypdf` | kb_ingest | извлечение текста из PDF |
| `python-docx` (`docx`) | kb_ingest | извлечение текста из DOCX |

**Внешние сервисы:**

- **SearXNG** — self-hosted мета-поисковик (JSON API на `127.0.0.1:8888`).
  Multi-engine (Yandex/DuckDuckGo/Bing/Brave) для устойчивости к троттлингу.
- **Tavily** — платный API извлечения текста (`api.tavily.com/extract`),
  последний tier каскада для high-trust источников.
- **LLM-агент через CLI** — внешний бинарь (`openclaw agent --json`), запускается
  через `subprocess`. Используется для суммаризации, генерации инсайтов и
  enrichment материалов. **Это и есть точка, которую надо заменить своей моделью.**
- **ru-fetch** — внешний бинарь (anti-bot fetcher через SOCKS5/xray-прокси с
  RU-IP и опциональным headless-браузером). Не входит в репозиторий.
- **Telegram Bot API** — опционально, для алертов/отчётов.

## Reuse — как переиспользовать

Код извлечён из большей системы, поэтому:

1. **Импорты-швы надо адаптировать.** Строки вида `from core.users import
   _find_user`, `from core.auth import _make_token`, `from web_app import ...`,
   `from templates import ...`, `import orchestrator` ссылаются на модули, которых
   здесь нет. Замени их своими реализациями или заглушками.

2. **LLM-вызовы идут через CLI-обёртку.** Везде, где есть
   `subprocess.run([OPENCLAW, "agent", ...])` / `subprocess.Popen([OPENCLAW, ...])`,
   модель вызывается как внешний процесс `openclaw`, который пишет JSON в stdout
   (а иногда результат — в файл). Парсинг ответа — в `core/llm_agent.py`
   (`_extract_reply`, `_parse_first_json_object`). Чтобы подключить свою модель,
   замени эти `subprocess`-вызовы на вызов своего SDK и верни тот же JSON-контракт.
   Подробности — в `docs/reuse-guide.md`.

3. **БД — SQLite со схемой в `schema/schema.sql`.** Путь к файлу БД задаётся в
   `core/config.py` (`KB_DB_PATH`). Доступ — через `core/kb_db.py`. Схема
   создаётся идемпотентно (`_kb_init`) с in-place миграциями.

4. **Жёстко зашитые абсолютные пути.** Многие скрипты содержат пути вида
   `/opt/newsapp/.openclaw/workspace/ops/r1_urls.json`,
   `/opt/newsapp/bypass/bin/ru-fetch`. Их надо вынести в конфиг или переопределить
   под свою среду.

5. **Trust-реестр доменов** (`TRUST_REGISTRY` в `r1_fetch_urls.py`, его зеркало в
   `core/archives.py`) — это конфиг доверенных источников, его можно взять и
   адаптировать под свой домен. Регулярки тем (`TRAVEL_RE`, `BANKING_RE`, `AI_RE`)
   — тоже чистый конфиг.

Гранулярные сценарии («хочу только сбор+скоринг», «только суммаризацию», «только
инсайты») разобраны в **`docs/reuse-guide.md`**.

## Настройка через `.env`

Скопируй `.env.example` → `.env` и заполни. Ключевые переменные:

- `TAVILY_API_KEYS` — ключи Tavily (через запятую), извлечение текста как fallback.
- `SEARXNG_URL` — адрес self-hosted SearXNG.
- `LLM_AGENT_CMD` / `LLM_AGENT_ID` — CLI LLM-агента и его профиль.
- `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID` — опционально, для доставки.
- `APP_DIR` — базовая директория для данных/БД/состояния.

> **Важно:** реальные секреты НИКОГДА не коммитятся. `.gitignore` уже исключает
> `.env`, `*.db`, `.tavily_keys`, `.jwt_secret`, `users.json`.

## Дисклеймер

Код предоставлен **как референс и для переиспользования отдельных частей**, «как
есть» (см. LICENSE / MIT). Это **не самодостаточный запускаемый продукт**: без
адаптации импортов-швов, замены CLI LLM-обёртки на свой вызов модели, настройки
путей и внешних сервисов (SearXNG, ru-fetch, Tavily) конвейер целиком не
запустится.

- **Секреты вычищены.** Реальные API-ключи, токены и пароли не должны
  присутствовать в публичной версии. См. раздел «Подозрения на секреты» в отчёте
  по результатам проверки — файл `pipeline/trendwatch_env.sh.example` требует
  ручной проверки перед публикацией (в извлечённой версии он содержал
  реалистичные значения).
- **Публичные доменные имена источников** (`sberbank.ru`, `vtb.ru`, `skift.com`,
  `reuters.com` и пр.) — это **конфигурация доверенных источников новостей**
  (trust-реестр), а не конфиденциальные данные.
- **Никита/Telegram chat_id `100000000`** в служебных скриптах — placeholder, не
  реальный идентификатор.
